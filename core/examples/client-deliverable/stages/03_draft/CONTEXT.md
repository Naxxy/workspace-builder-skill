## Inputs

### Layer 4 — Working artifacts (per-run)

- `../02_analysis/output/client-name_analysis.md` — The analysis from Stage 02, reviewed and approved by the human. Do not introduce findings or recommendations not present in the analysis.

### Layer 3 — Reference material (stable)

- `references/deliverable-format.md` — Structure, section headings, and length guidelines for the deliverable type this engagement requires.
- `../../_config/voice.md` — Voice and register conventions. All written output must conform to this guide.
- `../../_config/client-profile.md` — Client context, terminology, and communication preferences.

---

## Process

Read the analysis and all three reference files before writing.

Write the client-facing deliverable following the structure in `references/deliverable-format.md`. Use the client's terminology from the profile. Match the register and style from the voice guide.

Every finding and recommendation must come from the analysis. If you find that the analysis is missing something the deliverable structure requires, flag it rather than inventing content — the human can decide whether to add it or adjust the structure.

The deliverable must stand alone. A client who has not seen the intake summary or the analysis should be able to read it, understand the situation, and act on the recommendations.
