# Deliverable Format

Last updated: 2026-04-15

---

## Purpose

Defines the structure for the client-facing document produced in Stage 03. Update this file when the engagement requires a different format (report vs memo vs assessment). Do not modify it during a pipeline run.

---

## Default format: Advisory report

Use this structure unless the client profile specifies otherwise.

| Section | Content | Target length |
|---------|---------|---------------|
| Executive summary | Problem, key findings, top recommendation — the client should be able to act on this alone | 1 page max |
| Situation | What is true about the client's current situation, based on the analysis | 1–2 pages |
| Findings | Specific findings, in priority order | 1–3 pages |
| Recommendations | What to do, in priority order, each tied to a finding | 1–2 pages |
| Risks and considerations | What complicates the recommendations; what to watch for | 0.5–1 page |
| Next steps | Concrete actions with suggested owners and timelines | 0.5 page |

---

## Format rules

- Executive summary first — never last
- Each recommendation appears in the recommendations section; do not embed recommendations inside findings
- Use the client's terminology throughout — if the profile defines their terms, use them
- Avoid section titles that describe process ("Our approach", "Methodology") unless the client specifically requested this — they want outcomes, not process description

---

## Alternate formats

**Memo:** Executive summary + Findings + Recommendations only. No situation section. 2 pages max.

**Assessment:** Situation + Findings only. No recommendations — used when the client is forming their own view. Include a "Questions this raises" section instead of recommendations.

Update the section table above if switching to an alternate format.
